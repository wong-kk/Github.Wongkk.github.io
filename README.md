Git提交的三部曲
1.	git add
2.	git commit
3.	git push

git add
命令 git add -A 等同于git add . 

git commit
git commit –m”本次提交的的修改的备注”

git push
git push的情况相对比较复杂，分为以下几种情形
1.	第一次提交到本分支
2.	第2-次提交到本分支
3.	提交到其他分支


第一次提交到分支，命令比较长，命令为
git push origin main

第2-n次提交到本分支，命令可以简化为
git push
提交到其他分支
这个命令用的比较少，一般我们会切换到相应的分支并进行提交
如果我们需要提交到b分支，那么慢可以输入这个命令
git push origin b
